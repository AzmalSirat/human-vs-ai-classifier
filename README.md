# Distinguishing Human vs AI-Generated Bangla Text using BanglaBERT

A binary text classification project to detect whether Bangla text is written by humans or generated by AI models, with a focus on addressing domain shift caused by LLM evolution.

## Problem Statement

With the rapid proliferation of large language models (LLMs), distinguishing between human-written and AI-generated Bangla text has become increasingly important for:
- Academic integrity
- Content authenticity verification
- Journalism reliability
- Misinformation detection

Current detection models struggle with the evolution of AI writing styles. As newer LLMs (GPT-OSS 120B, GPT-4, Claude 2) produce increasingly human-like text compared to older models (GPT 3.5), models trained on older data show poor generalization.

## Dataset

### Overview
- **Total Samples**: 13,294 (merged_dataset.csv)
- **Class Distribution**: 
  - AI-generated: 6,854 (51.56%)
  - Human-written: 6,440 (48.44%)

### AI Data Breakdown
- ChatGPT: 3,681 (53.71%)
- Claude: 3,173 (46.29%)

### Human Data Breakdown
- Literature: 1,203 (18.68%)
- Newspapers: 5,237 (81.32%)
  - Prothom Alo, Samakal, Ittefaq, Daily Star, and others

### Mixed Stratified Approach
- **Proposed Dataset**: 5K samples from merged_dataset.csv (GPT-OSS 120B)
- **Reference Dataset**: 5K samples from literature paper dataset (GPT 3.5)
- **Total Training**: 10K samples with stratified 80/20 train/test split

## Methodology

### Model Architecture
- **Backbone**: csebuetnlp/banglabert (pre-trained Bangla BERT)
- **Task**: Binary text classification (Human vs AI-generated)
- **Input Handling**: 
  - Max sequence length: 256 tokens
  - Padding: max_length
  - Truncation: enabled

### Training Configuration
- **Loss Function**: Binary Cross-Entropy (BCE)
- **Optimizer**: Adam (learning rate: 2e-5)
- **Batch Size**: 8
- **Epochs**: 3
- **Training Framework**: HuggingFace Transformers (Trainer API)
- **Metrics**: Accuracy, Precision, Recall, F1-score

### Data Pipeline
1. **Data Preparation**: Merged dataset loading and cleaning
2. **Class/Category Filtering**: Removal of imbalanced categories
3. **Stratified Split**: 80/20 train/test maintaining class distribution
4. **Column Mapping**: Standardization to (text, label) format
5. **Tokenization**: BanglaBERT tokenizer with max_length=256
6. **PyTorch Formatting**: Conversion to PyTorch tensors
7. **Model Fine-tuning**: BanglaBERT classification head fine-tuning
8. **Evaluation**: Comprehensive metrics on both datasets
9. **Model Saving**: Full model and tokenizer preservation

### The Domain Shift Problem

Training on a single dataset shows strong performance on the same data but **significant degradation (20-26%) when tested on the other dataset**. This reveals the core issue: **Models overfit to dataset-specific LLM signatures and fail to generalize across different AI generations.**

### Final Model Performance (Mixed Stratified Dataset Approach)

#### Trained on Mixed Stratified Dataset (5K Proposed + 5K Reference)

**Tested on New Data (Proposed Dataset - GPT-OSS 120B)**
| Metric | Score |
|--------|-------|
| Accuracy | 96.00% |
| Precision | 93.98% |
| Recall | 98.29% |
| F1-Score | 96.09% |

**Tested on Old Data (Reference Dataset - GPT 3.5)**
| Metric | Score |
|--------|-------|
| Accuracy | 92.27% |
| Precision | 91.97% |
| Recall | 93.19% |
| F1-Score | 92.58% |

### Mixed Stratified Dataset Advantage

By training on a balanced combination of both datasets:
- **Maintains high accuracy** on new data: 96.00% (only 1.28% drop from single-dataset)
- **Dramatically improves cross-dataset performance**: 92.27% on old data (22% improvement vs single-dataset model)
- **Achieves consistent performance** across both LLM generations
- **Eliminates the 20-26% performance gap** observed in single-dataset models

## Key Findings

### 1. Domain Shift Challenge
- Models trained on reference data (GPT 3.5) show **high false positive rates** on newer AI text (GPT-OSS 120B)
- Indicates that AI has evolved significantly over 18 months
- Different LLM versions produce distinct stylistic signatures
- **Impact**: 20-26% accuracy drop when models trained on one dataset are tested on another

### 2. Mixed Dataset Advantage
- **22% improvement** in cross-dataset accuracy when using mixed training data
- Final model achieves **92.27% accuracy on reference data** (vs 70.74% for single-dataset model)
- Reduces overfitting to specific LLM generation patterns
- Improves generalization across different AI models
- **Result**: Consistent ~94-96% performance across both dataset types

### 3. Dataset Size vs Accuracy
- Accuracy improves progressively with dataset size
- **Saturation effect** observed: diminishing returns beyond 8,500-9,000 samples
- Optimal dataset size: ~10,000 samples for 96%+ accuracy

### 4. Token Length Optimization
- Optimal token length: ~100 tokens (~70 words per sample)
- Performance saturation observed beyond 100 tokens
- Dataset size of 8,500 with 100 tokens sufficient for 97% accuracy
